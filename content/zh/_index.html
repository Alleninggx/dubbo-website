Apache Dubbo 是一款支持多语言的、易用的 web 和 rpc 框架。 同时为构建企业级微服务提供服务发现、流量治理、可观测、认证鉴权等能力、工具与最佳实践。

Why Dubbo?


3.0 版本的正式发布，标志着 Apache Dubbo 正式进入云原生时代。3.0 在通信协议、服务发现、部署架构、服务治理上都对云原生基础设施进行了全面适配， 提供了Triple、应用级服务发现、Dubbo Mesh等核心特性。3.0 版本的正式发布，标志着 Apache Dubbo 正式进入云原生时代。3.0 在通信协议、服务发现、部署架构、服务治理上都对云原生基础设施进行了全面适配， 提供了Triple、应用级服务发现、Dubbo Mesh等核心特性。

{{% /blocks/lead %}} {{< blocks/section id="oceanNodes" >}} {{% blocks/feature image="flower" title="下一代RPC协议 - Triple" url="./docs/concepts/rpc-protocol/" %}} #### 易用性&多语言&编程范式 Triple 协议是基于 HTTP/2 之上定义的下一代 RPC 通信协议，相比于上一代 Dubbo2 协议，它具有更好的穿透性、通用性、以及面向网关代理场景的高性能表现， 提供了 Reactive Stream 数据交换模型。Triple 实现了对 gRPC 的完全兼容。 {{% /blocks/feature %}} {{% blocks/feature image="scalable" icon="fas fa-share-alt" title="应用级服务发现" url="./docs/concepts/service-discovery/" %}} #### 丰富的流量管控能力与可观测性 从服务/接口粒度到应用粒度的升级，使得 Dubbo 在集群可伸缩性、连接异构微服务体系上更具优势。应用粒度能以更低的资源消耗支持超百万实例规模集群； 实现与 Spring Cloud、Kubernetes Service 等异构微服务体系的互联互通。 {{% /blocks/feature %}} {{% blocks/feature image="blocks" icon="fa fa-share-alt" title="Mesh 解决方案" %}} #### 面向百万实例微服务集群&高性能通信 依赖 Dubbo Mesh 将 Dubbo 纳入 Istio 等开源 Mesh 治理体系， 3.0 同时支持 ThinSDK + Sidecar 与 FatSDK（Proxyless）的部署模式，为 Mesh 架构提供了更多选择，尤其对于面临迁移或混合部署的场景， Dubbo 提供了统一控制面的能力。 {{% /blocks/feature %}} {{% blocks/feature image="blocks" icon="fa fa-share-alt" title="Mesh 解决方案" %}} #### 企业生产环境验证 依赖 Dubbo Mesh 将 Dubbo 纳入 Istio 等开源 Mesh 治理体系， 3.0 同时支持 ThinSDK + Sidecar 与 FatSDK（Proxyless）的部署模式，为 Mesh 架构提供了更多选择，尤其对于面临迁移或混合部署的场景， Dubbo 提供了统一控制面的能力。 {{% /blocks/feature %}} {{< /blocks/section >}} {{< blocks/section id="video" background-image="kub_video_banner_homepage" >}}
